<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→`/studio` route | Untrusted browser requests enter the embedded Studio route |
| repo config→Sanity API | Public project and dataset identifiers are used to initialize remote content access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I | `sanity/env.ts` | mitigate | Read only `NEXT_PUBLIC_SANITY_PROJECT_ID`, `NEXT_PUBLIC_SANITY_DATASET`, and `NEXT_PUBLIC_SANITY_API_VERSION`; keep token handling out of Phase 1 files so no secrets are committed. |
| T-01-02 | D | `app/studio/[[...tool]]/page.tsx` | mitigate | Use the official catch-all route and `basePath: "/studio"` so nested Studio URLs do not 404 and deny editor access. |
| T-01-03 | T | `sanity/lib/client.ts` | mitigate | Centralize `createClient()` in one file and keep it separate from `lib/api.ts` so later query code does not drift into ad hoc clients with inconsistent config. |
</threat_model>
