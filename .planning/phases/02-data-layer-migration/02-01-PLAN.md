## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Next.js server → Sanity Content Lake | Server-side Work reads cross from trusted app code into remote CMS content |
| Route modules → compatibility transforms | Sanity documents are reshaped into the temporary legacy view model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | I | `sanity/lib/work.ts` | mitigate | Keep reads server-side with the shared published client only; do not introduce token-bearing browser fetches or preview perspective in Phase 2. |
| T-02-02 | T | Legacy compatibility transforms | mitigate | Validate optional legacy bridge fields before building the temporary project projection and fail closed on malformed required fields instead of guessing. |
| T-02-03 | D | `app/page.tsx`, `app/work/[slug]/page.tsx` | mitigate | Handle empty Sanity query results explicitly so builds and route generation fail predictably instead of crashing from undefined data. |
</threat_model>
